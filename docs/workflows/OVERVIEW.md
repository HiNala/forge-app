# Workflows overview

Forge’s first three flagship workflows are designed to feel like one product:

| Workflow | Best for | What you get |
|----------|-----------|--------------|
| **Contact / booking** | Local services, calendars, intake | Forms with validation, automations, booking semantics |
| **Proposal** | Contractors, agencies, quotes | Signable layouts, decision tracking, scoped follow-ups |
| **Pitch deck** | Fundraising, narrative updates | Slide-style pages, presenter mode, exports (rolling out) |

## When to use which

- Choose **contact / booking** when the goal is **scheduling or structured inquiries** synced to your calendar.
- Choose **proposal** when you need **priced scopes and acceptance** on a single client-facing URL.
- Choose **pitch deck** when you need **story + traction** in a presentable, repeatable format.

## Deep links

- Marketing pages link to `/signup?workflow=...` — after signup, onboarding can highlight the matching path and Studio opens with the same `?workflow=` hint.
- Command palette includes “New contact form / proposal / pitch deck” shortcuts for fast entry.
